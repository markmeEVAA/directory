Welcome to the EVAA Admin Portal

Use the tabs above (Manage Groups or Manage Members) to start editing. Click Need help? in the header any time to come back to this page.

Manage Groups tab

  • Group list: every EVAA / Fusion managed group. Click any row to drill in.
  • Group detail: see Directors (owners) and Members in two tables. + Add Director / + Add Member opens a user search; click a result to add. Or click + Create new user to onboard a brand-new user from scratch.
  • × button on a row: opens a smart remove dialog. You can choose Remove from this group only (just unlinks), Disable account only (preserve data) (soft offboard, keeps the license so mailbox + OneDrive stay alive; notifies admins by email), or Offboard fully (removes from all managed groups + removes license + disables account — 30-day data clock starts).

Manage Members tab

  • User list: all @evaasports.org and @avfusion.org users, sortable + filterable + paginated 25 per page. Disabled accounts shown in italic with a "disabled" badge.
  • User detail: see what groups the user belongs to (with Member / Owner / Member + Owner badges), their job title, account state (Active / Disabled), and assigned licenses.
  • Inline edit: the ✎ pencil next to the display name or job title lets you change it on the fly.
  • + Add to group: adds this user to any managed group as a member.
  • Offboard: opens the same 3-option dialog (preserve data vs full offboard).
  • Re-enable & re-license: appears only for disabled accounts. Recovers a user within the 30-day data window.

Creating new users

The + Create new user button (in either tab) opens a form that creates the M365 account, assigns the EVAA license, adds them to a chosen group, and sends them a welcome email with their sign-in info — all in one shot. They'll be forced to change the temp password on first sign-in.

Cross-navigation

Click any group name on the User Detail page → jumps to that group's detail. Click any person name on the Group Detail page → jumps to that user's detail. The tab indicator updates automatically.

Remove user

    "Preserve data" notifies portal admins by email so the group can decide together whether to fully offboard later or restore the user.

    What happens to their email and files? (read before deciding)

    Remove from this group only — only unlinks this one group. Nothing else changes. Account, license, mailbox, OneDrive all untouched.

    Disable account only (preserve data) — removes from all managed groups + disables the account + keeps the EVAA license assigned. The user can't sign in, but their mailbox and OneDrive are preserved indefinitely as long as the license stays assigned (~$3/mo cost continues). Notifies portal admins by email so the group can review later. Good for treasurers / departures where data may be needed.

    Offboard fully does three things instantly:

    1. Removes from all managed groups
    2. Removes the EVAA license
    3. Disables the account (can't sign in)

    ⚠ 30-day data window. Once the license is removed, Exchange starts a 30-day countdown on the user's mailbox and OneDrive. The account object itself stays in Entra as disabled forever, BUT:

    • Within 30 days: re-enable account + re-assign EVAA license → mailbox and OneDrive are fully restored. Group memberships need to be re-added manually.
    • After 30 days: Exchange auto-deletes the mailbox and OneDrive. The disabled user object remains but their data is gone. (Unless Litigation Hold or Retention Policies were configured — they are not, currently.)

    If you're not sure, pick Disable account only — the data stays safe and the portal admin group will get an email to discuss next steps.
