How this portal works

What this is

Self-service admin tool for managing M365 group memberships, board members, and user accounts. Restricted to members of the EVAA Portal Admins group. All actions write directly to Microsoft 365 via Graph — no approval routing for admin actions.

Manage Groups tab

  • Group list: every EVAA / Fusion managed group. Click any row to drill in.
  • Group detail: see Directors (owners) and Members in two tables. + Add Director / + Add Member opens a user search; click a result to add. Or click + Create new user to onboard a brand-new user from scratch.
  • × button on a row: opens a smart remove dialog. You can choose Remove from this group only (just unlinks) or Offboard fully (removes from all managed groups + removes the EVAA license + disables the account).

Manage Members tab

  • User list: all enabled @evaasports.org and @avfusion.org users, sortable + filterable + paginated 25 per page.
  • User detail: see what groups the user belongs to (with Member / Owner / Member + Owner badges), their job title, account state.
  • Inline edit: the ✎ pencil next to the display name or job title lets you change it on the fly.
  • + Add to group: adds this user to any managed group as a member.
  • Offboard fully: same 3-step removal as the group view but starting from the user.

Creating new users

The + Create new user button (in either tab) opens a form that creates the M365 account, assigns the EVAA license, and adds them to a chosen group — all in one shot. A temp password is generated; copy it and email it to the user. They'll be forced to change it on first sign-in.

Cross-navigation

Click any group name on the User Detail page → jumps to that group's detail. Click any person name on the Group Detail page → jumps to that user's detail. The tab indicator updates automatically.

Built with HTML/JS/MSAL/Graph. Source: github.com/markmeEVAA/directory/admin

    What happens to their email and files?

    Per-group remove — only unlinks this one group. Nothing else changes. Account, license, mailbox, OneDrive all untouched.

    Offboard fully does three things instantly:

    1. Removes from all managed groups
    2. Removes the EVAA license
    3. Disables the account (can't sign in)

    ⚠ 30-day data window. Once the license is removed, Exchange starts a 30-day countdown on the user's mailbox and OneDrive. The account object itself stays in Entra as disabled forever, BUT:

    • Within 30 days: re-enable account + re-assign EVAA license → mailbox and OneDrive are fully restored. Group memberships need to be re-added manually.
    • After 30 days: Exchange auto-deletes the mailbox and OneDrive. The disabled user object remains but their data is gone. (Unless Litigation Hold or Retention Policies were configured — they are not, currently.)

    If you might need someone's data preserved indefinitely (e.g. a treasurer leaving with financial records in their inbox), don't offboard fully — just remove from their groups and leave the account disabled-with-license. The license cost preserves the data.
